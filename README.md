<h1 align="center">UpdateSignatureForAdmins</h1>
<h3>Программка, которая позволяет создать подпись в профиле RoundCube</h3>
<h4>Написан для стандартной БД RoundCube</h4>
<h4>Для Для работы программы требуется конфигурационный файл, который следует расположить в папке "conf" в корне директории программы.
Если запустить программу без конфигурационного файла, то она сама создаст папку "conf" и завершит работу с ошибкой. Конфигурационный файл нужно создать в папке "conf" самостоятельно.</h4>
<p>Нужные для работы программы конфигурационные данные загружаются из конфигурационного файла config.txt, который находится в папке "conf" в корневой директории скрипта (если скрипт находится в папке "c:\\script", то путь до конфигурационного файла будет - "C:\\script\conf\config/txt" ).</p>
<p>Пример данных, которые должен содержать конфигурационный файл:</p>
<p style="padding-left: 30px;">
SQL_ServerName: srv-mail<br />
SQL_nameDB: RoundCubeMail<br />
SQL_login: SQLuser<br />
SQL_pass: VeryHardPassword<br />
LDAP_path: OU=OUname",DC=sstkvik,DC=local<br />
AD_atribut_pager_value: no_sig<br />
company: YourCompanyName<br />
streetAddress: ул. Сингапурское шоссе, 666<br />
city: г. Аристоград<br />
telephoneNumber: 8(800)555-35-35<br />
website: site.ru<br />
b2c: b2c.site.ru<br />
vkLink: vk.com/group<br />
okLink: ok.ru/group<br />
telegramLink: t.me/group<br />
rutubeLink: rutube.ru/channel/group/</p>

<p>Данные загружаются именно в такой последовательности. Если перепутать порядок, то в переменные скрипта попадут не верные данные и работа завершится с ошибкой.
Из каждой строки скрипт забирает информацию, которая записана после последовательности символов "двоеточие" и "пробел" - ": ". Данная последовательность служит разделителем между названием данных и непосредственно данными.</p>
<p>Например:</p>
<p style="padding-left: 30px;">
SQL_nameDB: RoundCubeMail<br />

"SQL_nameDB" - название данных, которое может быть любым и нужно для удобства чтения конфигурационного файа человеком.<br />
": " - разделитель, который отделяет название данных от самих данных.<br />
"RoundCubeMail" - непосредственно данные, которые "заберет" себе скрипт.</p>

<p>Скрипт берет из AD список sAMAccountName пользователей, получает по этим логинам значения следующих атрибутов:</p>
<p style="padding-left: 30px;">
	- cn;<br />- title;<br />
	- company;<br />
	- streetAddress;<br />
	- l;<br />
	- mail;<br />
	- telephoneNumber;<br />
	- mobile;<br />
	- pager.</p>
	
<p><br />Если атрибуты company, streetAddress, l, telephoneNumber пустые, то перезаписываются значениями, указанными в Config.txt</p>
<p>Отфильтровывает записи с пустым email и pager равным значению, которое указывается в Config.txt.</p>
<p>Выносит в отдельный список пользователей с одинаковым mail.</p>
<p>Из БД RoundCubeMail из таблицы identities достает список значений столбца email, и сравнивает со списком почтовых ящиков, полученных из AD.&nbsp;</p>
<p>Формирует список mail, котоые есть в identities, но нет в AD.</p>
<p>Формирует список mail, которые есть и в identities и в AD, но у них не совпадают cn и name (столбец в identities).</p>
<p>Формирует текст подписи для пользователей у которых cn и mail из AD совпадает с name и email из identities.</p>
<p>Если сформированный текст подписи не совпадает с тем, что храниться в identities, то скрипт перезаписывает его.</p>
<p>Все действия записываются в log\log - + "dd.MM.yyyy HH.mm.ss" + .txt.</p>
